# 5.1.2 Use Case Diagram

A Use Case Diagram is employed to model the interactions between users (actors) and the system, illustrating the different services the application will provide. For the USIU-A campus navigation system, the diagram identifies the primary actors and the use cases they are associated with. This ensures that the system functionalities remain user-centered and aligned with the needs established in the requirements phase.

> [Figure 13: Use Case Diagram]

Figure 13 above shows the use case diagram and it illustrates the key interactions between users and the Campus Navigation System. It identifies four primary actors: Student/Visitor, Admin, System (Automated Functions), and Maintenance Personnel.

The Student/Visitor interacts with the system to search for specific campus locations such as lecture halls, offices, cafeterias, and hostels. They can also view real-time navigation routes, access building details, check event or service locations, and receive notifications on temporary closures or route changes.

The Admin is responsible for managing the system database, which includes updating campus maps, adding or modifying location data, handling user accounts, and monitoring system performance. The admin also ensures that all navigation information remains accurate and up to date.

The Maintenance Personnel work closely with the admin to report or update information related to changes on the campus infrastructure—such as building renovations, new constructions, or blocked paths—to keep the navigation data reliable and current.

Meanwhile, the System (Automated Functions) performs background tasks such as providing route suggestions based on the user's current location, updating map data, sending notifications for map changes, and generating analytical reports on user activity.

The diagram also demonstrates various relationships such as «include» for shared processes like user authentication (login/sign-up) and location search, and «extend» for optional or conditional processes like route recalculation, event notifications, or dynamic updates triggered by real-time campus changes.
