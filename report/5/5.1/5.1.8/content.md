# 5.1.8 Class Diagram

A class diagram is a type of structural diagram in system design that illustrates the blueprint of an object-oriented system by showing the system's classes, their attributes, methods, and the relationships among them. It provides a static view of the system, helping developers understand how different components interact and depend on one another. In the Campus Navigation System, the class diagram serves as the foundation for understanding how data and functionality are organized within the system. It helps in identifying key entities such as users, locations, routes, and administrators, along with their specific roles and interactions. This visualization ensures that the system's architecture is well-structured, supports reusability, and maintains clear relationships between the different parts of the system.

> [Figure 21: Class Diagram]

Figure 21 shows the class diagram for the Campus Navigation System that depicts the main classes and their interconnections that support the system's functionality. Key classes include User, Admin, Location, Route, and NavigationSystem. The User class represents all users of the system, such as students and visitors, and includes attributes like user ID, name, and login credentials. The Admin class extends the User class, adding additional responsibilities like updating campus data, managing locations, and overseeing the system's operation.

The Location class stores information about various campus places such as lecture halls, offices, and facilities, while the Route class manages navigation details, including start and end points and the path generated between them. The NavigationSystem class serves as the central component that coordinates user requests, retrieves data from the Location and Route classes, and provides users with accurate navigation instructions. Relationships such as associations and dependencies are clearly shown — for example, a User interacts with the NavigationSystem to search for a location, and the system, in turn, communicates with the Route class to generate a navigation path.

Overall, the class diagram illustrates how different entities within the Campus Navigation System work together to support efficient campus guidance, data management, and user interaction, forming the backbone of the system's design and implementation.
