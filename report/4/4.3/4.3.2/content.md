# 4.3.2 Phase Two: System Development

## i. Frontend Development

**Objective:** To develop the user interface.

**Tasks:** Build mobile/web app interfaces for students, staff, and administrators using React.

**Deliverables:** Functional user interface with building search, interactive maps, and route display.

**Timeline:** Weeks 5–6

---

## ii. Backend Development

**Objective:** To implement business logic and core services.

**Tasks:** Develop backend APIs with Node.js/Express to handle authentication, map data management, and route generation.

**Deliverables:** Functional backend with secure APIs and navigation logic.

**Timeline:** Week 7

---

## iii. API Integration

**Objective:** To connect system components.

**Tasks:** Link frontend, backend, and PostgreSQL database (with PostGIS for spatial data) via REST APIs.

**Deliverables:** Fully integrated system with accurate data flow and route computation.

**Timeline:** Week 8

---

## iv. Deployment

**Objective:** To make the system accessible.

**Tasks:** Deploy the MVP of the Campus Navigation System on a cloud hosting service (e.g., AWS, Azure, or Heroku).

**Deliverables:** Hosted live application accessible to users.

**Timeline:** Week 9
