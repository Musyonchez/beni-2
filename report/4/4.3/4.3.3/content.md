# 4.3.3 Phase Three: Testing, Documentation, And Reporting

## i. Software Testing and Evaluation

**Objective:** To validate system functionality and performance.

**Tasks:** Perform unit testing (frontend, backend, database), integration testing (end-to-end routes), and user acceptance testing with USIU students and staff.

**Deliverables:** Testing report; validated navigation system ready for release.

**Timeline:** Weeks 10–11

---

## ii. Project Write-up

**Objective:** To document project outcomes.

**Tasks:** Prepare final report including research, methodology, system design, implementation, testing, and results.

**Deliverables:** Draft of final project report.

**Timeline:** Week 12

---

## iii. Final Presentation and Submission

**Objective:** To present findings and deliver final work.

**Tasks:** Conduct live demo of the USIU Campus Navigation System; prepare presentation slides; submit final project report.

**Deliverables:** Final report and presentation slides.

**Timeline:** Week 13
