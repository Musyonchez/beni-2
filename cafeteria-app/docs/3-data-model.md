# 3 — Firestore Data Model

All collections live at the root of the Firestore database.
Document IDs are auto-generated by Firestore unless noted otherwise.

---

## Collection: `users`
**Document ID:** Firebase Auth UID

| Field          | Type      | Notes                                      |
|----------------|-----------|--------------------------------------------|
| uid            | String    | Same as document ID                        |
| name           | String    | Full name                                  |
| email          | String    | Firebase Auth email                        |
| studentId      | String    | USIU student ID (e.g. "630400")            |
| role           | String    | "student" or "staff"                       |
| walletBalance  | double    | Current balance in KES (default 0.0)       |
| createdAt      | Timestamp |                                            |

**Java model:**
```java
public class User {
    private String uid, name, email, studentId, role;
    private double walletBalance;
    private com.google.firebase.Timestamp createdAt;
    // empty constructor + getters/setters
}
```

## Collection: `menuItems`
**Document ID:** auto

| Field       | Type      | Notes                                         |
|-------------|-----------|-----------------------------------------------|
| itemId      | String    | Same as document ID (set after creation)      |
| name        | String    | e.g. "Ugali & Beef Stew"                      |
| description | String    | Short description                             |
| price       | double    | In KES                                        |

| imageUrl    | String    | Firebase Storage URL or empty string          |
| available   | boolean   | true = orderable, false = greyed out          |
| createdAt   | Timestamp |                                               |

## Collection: `orders`
**Document ID:** auto

| Field             | Type      | Notes                                              |
|-------------------|-----------|----------------------------------------------------|
| orderId           | String    | Same as document ID                                |
| userId            | String    | Auth UID of the student                            |
| studentName       | String    | Denormalized for staff display                     |
| items             | List<Map> | Embedded (see orderItems schema below)             |
| status            | String    | "pending", "preparing", "ready", "collected"       |
| paymentMethod     | String    | "wallet" or "cash"                                 |
| totalAmount       | double    | Sum of (price × qty) for all items                 |
| estimatedWaitMin  | int       | Calculated at order creation time                  |
| createdAt         | Timestamp |                                                    |

### Embedded `items` list (each map):
| Field    | Type   | Notes              |
|----------|--------|--------------------|
| itemId   | String |                    |
| name     | String | Denormalized       |
| price    | double | Price at order time|
| quantity | int    |                    |

**Java model:**
```java
public class Order {
    private String orderId, userId, studentName, status, paymentMethod;
    private List<OrderItem> items;
    private double totalAmount;
    private int estimatedWaitMin;
    private com.google.firebase.Timestamp createdAt;
}

public class OrderItem {
    private String itemId, name;
    private double price;
    private int quantity;
}
```

## Collection: `preOrders`
**Document ID:** auto

| Field         | Type      | Notes                                                  |
|---------------|-----------|--------------------------------------------------------|
| preOrderId    | String    | Same as document ID                                    |
| userId        | String    | Auth UID                                               |
| studentName   | String    | Denormalized                                           |
| mealSlot      | String    | "lunch" or "dinner"                                    |
| items         | List<Map> | Same schema as orders.items                            |
| totalAmount   | double    |                                                        |
| scheduledDate | String    | "YYYY-MM-DD" — the date this pre-order is for          |
| recurring     | boolean   | true if it repeats weekly                              |
| recurringDays | List<String> | e.g. ["monday", "thursday"] — only if recurring==true |
| status        | String    | "scheduled", "confirmed", "cancelled"                  |
| createdAt     | Timestamp |                                                        |

**Note:** When a recurring pre-order is processed at cut-off, Cloud Functions create a new `preOrders` document for the next occurrence and set the current one to "confirmed" or "cancelled".

## Collection: `walletTransactions`
**Document ID:** auto

| Field         | Type      | Notes                                           |
|---------------|-----------|-------------------------------------------------|
| txId          | String    | Same as document ID                             |
| userId        | String    | Auth UID of the student                         |
| type          | String    | "topup" or "deduction"                          |
| amount        | double    | Always positive; sign implied by type           |
| description   | String    | e.g. "Top-up by staff", "Order #abc123", "Pre-order #xyz — lunch" |
| relatedOrderId| String    | orderId or preOrderId, or empty string for top-ups |
| createdAt     | Timestamp |                                                 |

## Firestore indexes needed
- `orders` — composite index: `userId` ASC + `createdAt` DESC (for student order history)
- `orders` — composite index: `status` ASC + `createdAt` ASC (for staff orders list)
- `preOrders` — composite index: `userId` ASC + `createdAt` DESC
- `preOrders` — composite index: `mealSlot` ASC + `scheduledDate` ASC + `status` ASC (for Cloud Functions query)
- `walletTransactions` — composite index: `userId` ASC + `createdAt` DESC

Create these in Firebase Console → Firestore → Indexes, or let the SDK error message generate the link automatically.
