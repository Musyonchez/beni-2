# 4.3.3 Phase Three: Testing, Documentation, And Reporting

## i. Software Testing and Evaluation

**Objective:** To validate system functionality, performance, and security.

**Tasks:** Execute all 20 defined test cases covering order flow, wallet atomicity, pre-order cut-off accuracy, real-time sync, role-based access, and performance; conduct UAT with 5 students and 2 cafeteria staff; document results.

**Deliverables:** Testing report with pass/fail results for all 20 test cases; UAT feedback summary.

**Timeline:** Weeks 11–12

---

## ii. Project Write-up

**Objective:** To document all project outcomes.

**Tasks:** Prepare final report including research, system design diagrams, implementation details, testing results, challenges encountered, and conclusions.

**Deliverables:** Draft of final project report.

**Timeline:** Week 12

---

## iii. Final Presentation and Submission

**Objective:** To present findings and deliver final work.

**Tasks:** Conduct live demo of the USIU Cafeteria Ordering System covering ordering, pre-scheduling, wallet management, and staff operations; prepare presentation slides; submit final report.

**Deliverables:** Final report and presentation slides submitted.

**Timeline:** Week 13
