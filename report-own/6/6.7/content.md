# 6.7 Conclusion

This chapter presented the complete implementation of the USIU Cafeteria Ordering System, documenting the construction, integration, and deployment of all components across the two client applications and the cloud backend. The Android application was built in Java using XML layouts and Material Design 3 components, structured around a single-activity MVVM architecture with a centralised FirestoreRepository handling all data access. The web-based admin panel was built with Next.js 14 and TypeScript, using the Firebase Admin SDK in server-side API routes to securely manage staff accounts and system data. Both applications share the same Firebase project and Firestore database, governed by a unified set of security rules that enforce role-based access for students, staff, and administrators.

The backend relies on Firebase Firestore for real-time data synchronisation, Firebase Authentication for identity management across all three roles, and Firebase Cloud Messaging for push notification delivery. Supabase Edge Functions written in TypeScript replace the Firebase Cloud Functions that the original design specified, enabling the scheduled pre-order cut-off automation and push notification dispatch to operate on the free Spark plan without requiring a Firebase Blaze subscription. The critical technical insight that Deno's gRPC incompatibility requires the Firebase Admin SDK to be configured with preferRest: true was discovered and resolved during implementation, and is documented in the code and in the deployment notes.

All wallet operations — top-ups by staff, deductions for regular orders, and pre-order charges at cut-off — execute inside Firestore runTransaction calls, providing atomic, consistent, and isolated updates to wallet balances regardless of concurrent access from multiple devices. The hire/show fragment navigation pattern preserves UI state across tab switches, and the FirestoreRepository's snapshot listeners ensure that order status changes propagate to all affected screens in real time. Deployment was achieved entirely on free-tier infrastructure: Firebase Spark, Supabase free, cron-job.org free, and Vercel free, resulting in a production-ready system with zero recurring operational costs.
