# 6.1 Introduction

The implementation phase of the USIU Cafeteria Ordering System represented the crucial transformation of the theoretical design specifications and comprehensive requirements documentation into fully operational, production-ready software. This development stage encompassed the systematic construction of two distinct client applications — a native Android mobile application and a web-based admin panel — along with the configuration of a Firebase cloud backend and two Supabase Edge Functions that handle automated pre-order processing and push notification dispatch.

The primary objective was to deliver a robust, intuitive, and feature-complete cafeteria ordering solution addressing the needs of three user roles: students who browse the menu, place orders, and manage pre-orders and wallet balances; cafeteria staff who process incoming orders and handle student wallet operations; and administrators who manage the menu catalogue and create staff accounts. Special emphasis was placed on ensuring that all payment operations — wallet deductions, top-ups, and pre-order charges at cut-off time — execute atomically using Firestore transactions, preventing any inconsistency in wallet balances under concurrent access.

The implementation prioritised simplicity and reliability over complexity. The Android app follows a single-activity architecture with a BottomNavigationView and fragment-based navigation using hide/show transactions to preserve scroll state. All Firestore access is centralised in a single FirestoreRepository class, and UI state is managed through ViewModel and LiveData classes following the MVVM pattern. The admin panel uses Next.js 14 App Router with server-side Firebase Admin SDK calls to securely create staff accounts and manage Firestore data. Both applications share the same Firebase project and Firestore database, governed by a unified set of security rules that enforce role-based access at the document level.
