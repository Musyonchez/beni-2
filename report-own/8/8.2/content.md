# 8.2 Achievements

Significant milestones were accomplished during the development of the USIU Cafeteria Ordering System, spanning all three application tiers and covering the full scope of the original requirements.

**Complete three-role system across two applications.** The most substantial achievement was delivering a fully functional system serving three distinct user roles. Students and cafeteria staff use a native Android app — the student interface provides five tabs (Menu, Cart, Orders, Pre-orders, Profile/Wallet) while the staff interface provides three tabs (StaffOrders, StaffMenu, StaffWallet). Administrators use a separate Next.js web panel for menu and user management. Role-based routing from `LoginActivity` ensures that each Android user lands on the correct interface after authentication, with no manual configuration required.

**Real-time order lifecycle with FCM notifications.** The end-to-end order flow — from a student placing an order to receiving a push notification when the meal is ready — was implemented and verified on physical devices. Firestore snapshot listeners provide sub-second status propagation between the student and staff interfaces, and Firebase Cloud Messaging delivers the "order ready" notification reliably within approximately 3 seconds. This replaces all verbal communication between student and counter staff for routine orders.

**Atomic in-app wallet with full audit trail.** The Firestore `runTransaction` implementation for wallet order placement atomically decrements the student balance, creates the order document, and records the transaction in a single operation with no risk of partial writes. The staff wallet top-up and deduction operations are equally atomic, and every transaction records the `staffId` of the staff member who performed it, providing a complete financial audit trail without a separate accounting system.

**Pre-order scheduling with automated cut-off processing.** The pre-order system — allowing students to schedule wallet-funded meals in advance with a 10:00 AM (lunch) or 5:00 PM (dinner) cut-off — was fully implemented and deployed. The Supabase Edge Function `process-cutoff`, triggered by cron-job.org at the scheduled times, queries all pending pre-orders, processes wallet deductions atomically, dispatches FCM confirmations or cancellation notices, and schedules recurring pre-orders for the next occurrence. This server-side automation required no persistent server and runs entirely on the free tier.

**Firestore security rules and composite indexes.** Five composite Firestore indexes and a comprehensive set of security rules with role-based helper functions (`isAuth()`, `isOwner()`, `isStaff()`, `isAdmin()`, `isPrivileged()`) were deployed to production. The rules correctly enforce data partitioning — students can only read their own orders, staff can read all orders, and the `walletBalance` field is write-protected against client-side modification. All three security test cases confirmed these rules operate as specified.

**Next.js admin web panel deployed on Vercel.** A separate Next.js 14 web application provides administrators with a dashboard for menu management, staff account creation (via the Firebase Admin SDK), and a directory of all registered users. The panel is deployed on Vercel's free tier with HTTPS, uses server-side Firebase Admin SDK calls exclusively, and requires no Android device access for administrative operations.

**Free-tier cloud infrastructure.** The entire system — Firebase Firestore, Firebase Auth, FCM, Supabase Edge Functions, cron-job.org scheduling, and Vercel hosting — operates within free service tiers, incurring zero monthly cost. This demonstrates that production-grade, multi-tier university software can be built and deployed without infrastructure budget, which is a meaningful outcome for resource-constrained institutions.
