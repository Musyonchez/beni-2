# 5.4 Conclusion

The Cafeteria Ordering System is a project designed to address the real challenges of food ordering and queue management at the United States International University Africa (USIU-Africa) cafeteria. This chapter has presented the complete system design, covering the system models, functional requirements, and non-functional requirements that together define both what the system must do and the quality standards it must meet.

The system models section documented the architecture of a three-tier Android and Firebase solution and illustrated its behaviour through a comprehensive set of diagrams: use case, entity-relationship, data flow, flowchart, activity, sequence, class, and wireframe diagrams. These diagrams confirm that the system is well-structured, with clear separation between the student-facing mobile application, the Firebase backend services, and the staff management interface.

The functional requirements capture seventeen specific capabilities spanning student ordering (browse, cart, regular order, pre-order, recurring schedule), real-time estimated wait time, staff management (order processing, menu management, wallet top-up), and system-level automation (pre-order cut-off via Cloud Functions, FCM push notifications, atomic Firestore transactions, and a complete wallet audit trail). The non-functional requirements establish the performance, security, usability, reliability, scalability, maintainability, and compliance standards that the implementation must satisfy.

In conclusion, the analysis and design presented in this chapter provide a solid and feasible blueprint for the Cafeteria Ordering System. The combination of a modern Android front end, a scalable Firebase backend, and automated pre-order processing through Cloud Functions distinguishes this system from conventional point-of-sale or manual queuing approaches. With this design in place, the implementation phase, covered in the following chapter, can proceed with clarity and confidence.
