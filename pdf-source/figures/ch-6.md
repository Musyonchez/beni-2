## Figure 33: Supabase Edge Function (process-cutoff) — logic flow

```plantuml
@startuml Figure33_ProcessCutoffLogic
skinparam backgroundColor white
skinparam defaultFontName Arial
caption Figure 33: process-cutoff Edge Function — Logic Flow

start
:Receive HTTP POST from cron-job.org\n(mealSlot: "lunch" or "dinner");
if (Authorization header valid?) then (yes)
  :Query preOrders\n(mealSlot = target, status = scheduled,\nscheduledDate = today);
  while (More pre-orders to process?) is (yes)
    :Read users/{userId}.walletBalance;
    if (balance >= preOrder.totalAmount?) then (yes)
      :runTransaction()\n— deduct walletBalance\n— update preOrder.status = confirmed\n— write walletTransactions document;
      :Send FCM confirmation notification\nto student device;
    else (no)
      :Update preOrder.status = cancelled;
      :Send FCM cancellation notification\n("Insufficient funds — pre-order cancelled");
    endif
    if (preOrder.recurring == true?) then (yes)
      :Create new preOrders document\n(next week's date, status = scheduled);
    endif
  endwhile (no more)
  :Return HTTP 200 with processing summary\n(confirmed count, cancelled count);
  stop
else (no)
  :Return HTTP 401 Unauthorised;
  stop
endif
@enduml
```

## Figure 37: Pre-order cut-off integration — end-to-end automated pipeline

```plantuml
@startuml Figure37_PreOrderCutoffIntegration
skinparam backgroundColor white
skinparam defaultFontName Arial
caption Figure 37: Pre-order Cut-off Integration — end-to-end flow

participant "cron-job.org" as CRON
participant "process-cutoff\n(Supabase Edge Function)" as EF
participant "Firestore" as FS
participant "FCM" as FCM
participant "Student Device" as STU

CRON -> EF : HTTP POST (07:00 UTC — mealSlot: lunch)\nor HTTP POST (14:00 UTC — mealSlot: dinner)
EF -> FS : query preOrders\n(mealSlot = target, status = scheduled,\nscheduledDate = today)
FS --> EF : [preOrder1, preOrder2, ...]

loop for each preOrder
  EF -> FS : runTransaction(): read walletBalance
  alt balance >= totalAmount
    EF -> FS : deduct balance\nstatus = confirmed\nwrite walletTransactions
    EF -> FCM : sendNotification(fcmToken, "Pre-order confirmed")
  else balance < totalAmount
    EF -> FS : status = cancelled
    EF -> FCM : sendNotification(fcmToken, "Insufficient funds")
  end
  opt recurring == true
    EF -> FS : create next-week preOrders document\n(scheduledDate += 7 days, status = scheduled)
  end
end

FCM --> STU : push notification (confirmed or cancelled)
EF --> CRON : HTTP 200 { confirmed: N, cancelled: M }
@enduml
```

```plantuml
participant "OkHttp (background thread)" as HTTP

REPO -> HTTP : HTTP POST\nhttps://<project>.supabase.co/functions/v1/notify-order-ready\nAuthorization: Bearer BuildConfig.FUNCTIONS_SECRET\nContent-Type: application/json\n{ orderId, userId }
HTTP -> EF : request (background thread — does not block UI)
EF -> EF : verify Authorization header
EF -> FS : read users/{userId}.fcmToken
FS --> EF : fcmToken
EF -> FCM : messaging().send(token, payload)
FCM --> EF : message ID
EF --> HTTP : HTTP 200 { status: "sent" }
HTTP --> REPO : response (ignored — fail-silent)
note over REPO, HTTP
  Call is made after the Firestore status
  write succeeds. Network failure does not
  affect the order status update.
end note
@enduml
```
