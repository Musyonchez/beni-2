## Project Phases and Tasks

### 1. Phase One: Research and Requirements Gathering

This initial phase established the foundation of the Campus Navigation System by focusing on research, requirements gathering, proposal development, and system design. The tasks in this phase followed a structured sequence to ensure that the project was based on accurate data and user needs. The process began with comprehensive research to understand existing campus navigation solutions and identify gaps in accessibility, usability, and real-time location tracking. Following this, requirements gathering was carried out to define the system's functional and non-functional needs based on student, staff, and visitor feedback. Proposal development then formalized the system's objectives, scope, and implementation strategy. Finally, system design was undertaken, where the architecture, data flow, and navigation logic were defined. The successful completion of these tasks led to **Milestone 1: Research and System Design Completed**, marking the project's readiness to move into the development stage.

### 2. Phase Two: System Implementation

This phase concentrated on building and integrating the core components of the Campus Navigation System. It began with frontend development, where the user interface and user experience were created to allow smooth map navigation and intuitive access to building details. This was followed by backend development, which established the server-side logic, handled database connections, and supported real-time location queries. Next, API integration was performed to connect the system with mapping services such as Google Maps API and indoor positioning tools. Once the core functionalities were in place, deployment was done to host the system in a live environment, making it accessible for testing and feedback. The completion of this stage marked **Milestone 2: System Implementation Completed**, signifying that the Campus Navigation System was functional and ready for evaluation.

### 3. Phase Three: Testing, Documentation, and Presentation

The final phase emphasized ensuring system reliability, preparing documentation, and presenting the project outcomes. It began with software testing and evaluation, where the system was assessed for accuracy in navigation, performance, and user experience. Any detected bugs or inconsistencies were fixed to ensure stable operation. Following testing, project documentation was completed, summarizing all development processes, tools used, challenges encountered, and how they were addressed. The phase concluded with the final presentation and report submission, where the system was demonstrated to stakeholders and evaluators. This marked **Milestone 3: Testing, Documentation, and Presentation Completed**, confirming that the Campus Navigation System met its objectives and was ready for deployment within the university environment.

## Dependencies

The project followed a logical flow of dependencies that ensured each task was completed before the next began. For instance, system design could only start after requirements gathering was finalized. Similarly, backend development depended on the completion of frontend structures, while API integration required both frontend and backend to be in place. Testing and documentation were dependent on the successful deployment of the system. These dependencies ensured a smooth transition between activities, reduced bottlenecks, and maintained consistent project progress.

## Milestones

The project's milestones represented significant checkpoints that helped track progress and ensure the timely completion of critical stages. Milestone 1 marked the end of the research and system design phase, providing a validated blueprint for development. Milestone 2 represented the completion of system implementation, with all core functionalities successfully built and integrated. Finally, Milestone 3 indicated the completion of testing, documentation, and presentation, confirming that the system was fully functional, tested, and ready for use. These milestones provided structure, accountability, and clear progress tracking throughout the development of the Campus Navigation System.
