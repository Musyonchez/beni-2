# 8.3 Challenges Encountered

Although the USIU Campus Navigator was successfully built and deployed, the development process presented several technical and functional hurdles that required rigorous problem-solving. One of the primary difficulties was ensuring the persistence of the map state during navigation transitions. Early in the development phase, switching between the "Map" and "Facilities" tabs caused the Google Map instance to reload, resetting the user's camera position and zoom level. This disrupted the user experience significantly. Resolving this required a shift in the fragment management strategy, moving from simple replacement transactions to a more complex hide-and-show logic within the MainActivity, ensuring the map instance remained active in the background.

Another complex challenge involved managing the Camera Boundary Restrictions to keep the user focused on the campus. Determining the precise latitude and longitude coordinates for the Southwest and Northeast corners of USIU required extensive testing. Initially, the bounds were too tight, preventing users from viewing the outskirts of the campus, or too loose, allowing them to scroll into neighboring estates. Fine-tuning these coordinates and configuring the setLatLngBoundsForCameraTarget method required multiple iterations to achieve a balance between usability and restriction.

Implementing the "Smart Status" logic also posed logical challenges, particularly regarding edge cases in the operating schedule. The initial algorithm struggled to correctly handle weekends for administrative offices, occasionally marking them as "Open" on Saturdays. This required rewriting the Calendar logic to include strict day-of-week checks alongside hour-of-day comparisons. Additionally, handling Runtime Permissions for GPS access proved intricate across different Android versions. Ensuring that the application did not crash if a user denied location permissions required implementing robust fallback logic that allowed the app to function in "Manual Mode" without the blue location dot.

Finally, ensuring UI responsiveness across different device screen sizes presented a significant design challenge. The facility cards in the directory occasionally rendered with overlapping text on smaller, older devices. This required optimizing the XML layouts using ConstraintLayout and defining flexible dimensions rather than fixed pixel sizes. Despite these issues, persistent debugging, field testing on physical devices, and code optimization helped overcome these setbacks. The result was a stable, resilient application capable of supporting reliable campus navigation for all users.
