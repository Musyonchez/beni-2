# 7.1 Introduction

This chapter outlines the comprehensive testing and evaluation efforts conducted on the USIU Campus Navigator to ensure its robustness, accuracy, and usability. The testing phase was essential in validating the performance of key modules, such as interactive map rendering, facility searching, navigation routing, and user authentication. It focused on confirming that all features functioned correctly and securely across different user roles, including students, faculty, and administrators.

A variety of testing strategies were applied, including functionality testing to verify individual features, interface testing to assess responsiveness and user interactions, and location-based testing to ensure the Google Maps API integration worked precisely within campus boundaries. Additional evaluations examined the system's ability to maintain data consistency in the local database, handle simultaneous navigation requests, and enforce proper access restrictions. The insights gained from these processes guided refinements and ensured the system met performance expectations for reliable campus navigation and facility discovery.

In addition to these core evaluations, usability testing was performed to gauge how intuitively users could navigate the system and perform common tasks such as searching for a building, generating a route, or filtering facility categories. Feedback from actual users during simulated usage scenarios helped identify minor UI inefficiencies and areas for clarity improvement, which were promptly addressed. Moreover, compatibility testing was carried out across various Android devices and screen sizes to confirm the system's consistent behavior and visual integrity. These iterative testing cycles ensured that the USIU Campus Navigator not only met its technical requirements but also delivered a smooth and accessible user experience suitable for the university community.
