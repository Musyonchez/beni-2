# 5.1.6 Activity Diagram

An activity diagram is a type of behavioral diagram used in system modeling to represent the flow of activities and actions within a process. It focuses on how different tasks are carried out in sequence, the decisions made along the way, and how various activities connect to achieve a specific goal. In the context of the Campus Navigation System, the activity diagram visually demonstrates the step-by-step workflow from when a user interacts with the system to when they receive navigation results or updates. It helps illustrate both user-driven and system-driven processes, such as logging in, searching for locations, generating routes, and updating campus data. By using decision points, parallel activities, and start/end states, the activity diagram gives a clear view of the logical flow of operations and how different parts of the system collaborate to deliver smooth navigation and user support.

> [Figure 19: Activity Diagram]

Figure 19 shows the activity diagram for the Campus Navigation System that outlines the logical sequence of actions that occur when users interact with the platform. The process begins with the user login or registration, after which the system verifies credentials. Once authenticated, the user can perform actions such as searching for a location, viewing campus routes, or requesting navigation assistance. The diagram shows how these actions flow from one stage to another — for instance, after entering a search query, the system retrieves relevant location data, processes it through the navigation logic, and displays the optimal route.

For administrative users, the activity diagram includes additional actions such as adding or updating campus locations and managing user data. Decision nodes are used to represent choices, such as whether the entered location exists in the database or whether an update is required. If conditions are met, the process continues; otherwise, the system prompts the user to try again or provides alternative options.

Overall, the activity diagram provides a detailed visualization of how the Campus Navigation System handles both normal and exceptional scenarios. It emphasizes the smooth flow of data and decisions from start to finish, ensuring efficient user interactions and accurate delivery of navigation information across different user roles within the system.
