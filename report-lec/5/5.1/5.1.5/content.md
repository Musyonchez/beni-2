# 5.1.5 Flowchart

A flowchart is a visual representation that outlines the step-by-step flow of processes or activities within a system. It uses standardized symbols such as ovals, rectangles, diamonds, and arrows to depict actions, decisions, and the direction of process flow. In the context of the Campus Navigation System, the flowchart provides a clear and logical view of how users interact with the system from start to finish. It visually demonstrates the sequence of operations — from user login and input of location queries to the generation of navigation routes and display of results. This makes it easier to understand the logical order of events, identify decision points, and visualize how the system responds under different user actions or conditions.

> [Figure 18: Flowchart]

Figure 18 shows the flowchart of the Campus Navigation System that illustrates the systematic process a user follows when using the application. It begins when the user launches the system and logs in, after which the system verifies the credentials. If the login is successful, the user can proceed to search for a specific location or building within the campus. Once the search query is entered, the system checks the Campus Location Database for a match. If the location exists, the system generates and displays the most efficient navigation route. However, if the location is not found, an error message is shown prompting the user to refine their search.

Additionally, the flowchart demonstrates how administrators and security officers interact with the system to update or modify campus information. The process ensures that new data is stored and reflected in real-time for all users. By visually presenting each step, the flowchart helps to clarify the logical flow of operations, simplifies system understanding, and supports the development and troubleshooting processes. It effectively captures how different users navigate through the system, making it a valuable tool for system design, documentation, and future improvements.
